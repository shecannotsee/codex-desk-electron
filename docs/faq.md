# FAQ

## 1. 为什么提示“未找到 codex 命令”？

原因：系统 PATH 中没有可执行的 `codex`。

处理：

1. 在终端执行 `codex --version` 确认可用。
2. 确认 GUI 使用的命令配置正确（`Codex命令`）。

## 2. 为什么发送时报“目录不存在”？

原因：当前配置的 `工作目录` 不存在或不可访问。

处理：

1. 切换到存在的目录。
2. 确保该目录权限正常。

## 3. 为什么模型显示为未知或获取失败？

模型获取是多路探测（命令参数 / 配置文件 / 运行 JSON / 主动探测）。
任一路失败都可能导致最终不可用。

处理：

1. 先在命令里显式加 `--model`。
2. 再尝试菜单“运行 -> 获取模型”。

## 4. 运行中关闭窗口会怎样？

现在会弹确认框：

1. 取消
2. 停止任务并关闭
3. 直接关闭

建议优先选“停止任务并关闭”。

## 5. 为什么有些按钮不在页面里了？

部分操作已迁移到顶部菜单，减少页面噪音。
例如：清空运行日志、刷新版本/模型、面板显隐。

## 6. “视图缩放”和“对话字号”是什么关系？

两者独立控制，最终效果叠加：

- 视图缩放：窗口级
- 对话字号：内容级

## 7. 运行步骤为什么默认折叠？

为了在长任务里保持可读性。可按步骤单独展开。

## 8. 我在 Windows/macOS 上可以直接用吗？

当前文档状态是：

- 已验证：`Ubuntu 22.04`
- 未验证：`Windows`、`macOS`

不是明确不支持，而是还未完成系统化验证。

## 9. 数据存在哪里？

默认在仓库目录下：

- `.codexdesk/state.electron.json`

并兼容读取旧路径：

- `~/.codexdesk/state.electron.json`
