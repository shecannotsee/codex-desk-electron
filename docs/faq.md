# FAQ

## 1. 为什么提示“未找到 codex 命令”？

原因：系统 PATH 中没有可执行 `codex`。

处理：

1. 终端执行 `codex --version`。
2. 检查 `Codex命令` 配置是否可执行。

## 2. 为什么发送时报“目录不存在”？

原因：当前 `工作目录` 不存在或不可访问。

处理：

1. 切换到存在目录。
2. 确认目录权限。

## 3. 为什么模型显示未知或获取失败？

模型探测为多路合并（命令参数 / 配置文件 / 运行 JSON / 主动 probe）。

处理：

1. 命令中显式加 `--model`。
2. 在“设置 -> 运行 -> 获取模型”刷新。

## 4. 为什么终端里能 `git clone` / `ssh`，Codex Desk 里不行？

常见原因：

1. 桌面应用启动时没有继承终端登录 shell 的环境变量，例如 `SSH_AUTH_SOCK`、`PATH`、代理变量。
2. 图形界面的默认 `codex exec` 参数比终端里更保守。

当前版本会在运行 `codex` 子进程前导入登录 shell 环境，并默认使用：

```bash
codex exec --skip-git-repo-check --dangerously-bypass-approvals-and-sandbox
```

如果仍失败，请先在终端里确认：

```bash
echo "$SSH_AUTH_SOCK"
which codex
ssh -T git@github.com
```

## 5. 运行中关闭窗口会怎样？

会弹确认框：

1. 取消
2. 停止任务并关闭
3. 直接关闭

建议优先选“停止任务并关闭”。

## 6. 为什么页面上少了以前的一些按钮？

功能被整理到两个入口：

1. 左侧会话右键菜单（新建/重命名/关闭）。
2. 右上角“设置”多级菜单（对话/运行/视图/编辑/应用/帮助）。

## 7. “视图缩放”和“对话字号”是比例联动吗？

不是。两者独立，最终视觉效果叠加：

- 视图缩放：窗口级
- 对话字号：内容级

## 8. 运行步骤为什么默认折叠？

为了在长任务中保持可读性。可逐条展开查看。

## 9. 排队的消息在哪看？

切到右侧“运行步骤”标签，顶部有“待执行排队消息”面板，显示内容、顺序和入队时间。

## 10. 怎么切换浅色/深色？

“设置 -> 视图 -> 主题”。

窗口顶部区域也会同步切换，不需要单独设置。

## 11. 左侧会话能调宽度吗？

可以。拖动左侧会话区与主区之间的分隔线即可，宽度会持久化保存。

## 12. 输入框字多了会自动换行吗？

会。输入框使用多行 `textarea`，会按边界自动换行。

未发送内容会按会话保存在本地；切换会话或关闭窗口后重新打开时会恢复。

## 13. 我在 Windows/macOS 上可以直接用吗？

当前状态：

- 已验证：`Ubuntu 22.04`
- 未验证：`Windows`、`macOS`

## 14. 数据存在哪里？

- 会话状态：`.codexdesk/state.electron.json`
- 兼容旧目录：`~/.codexdesk/state.electron.json`
- UI 偏好：浏览器本地存储 `codexdesk.ui-prefs.v1`

## 15. Ubuntu 上怎么安装成桌面应用？

使用 `.deb`：

1. 在 `src` 执行 `npm run dist:deb`
2. `sudo apt install ./dist/Codex\ Desk-*.deb`
3. 应用菜单搜索 `Codex Desk` 或执行 `codex-desk`

详见：`docs/deploy-ubuntu.md`。

## 16. Ubuntu 上怎么完整卸载？

详见：`docs/uninstall.md`。

## 17. 运行中聊天区为什么会出现临时状态块？

这是 GUI 的运行中提示区，不是最终消息。

- 第一行显示“正在执行中...”
- 第二行显示当前步骤
- 最终回复到达后会自动消失

如果步骤内容太长，只会在状态块内部滚动，不会持续把整段聊天顶到最底部。
