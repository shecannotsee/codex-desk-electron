# 用户指南（按场景）

> 本文按“功能目标 / 操作步骤 / 参数配置 / 错误边界 / CLI 关系”统一描述。

## 场景 1：创建会话并发送消息

### 功能目标

快速开始一轮 Codex 对话，并保留历史消息。

### 操作步骤

1. 点击“新建对话”。
2. 在输入框输入内容。
3. 按 `Ctrl+Enter` 或点击“发送”。

### 参数/配置说明

- `Codex命令`：默认 `codex exec --skip-git-repo-check`
- `工作目录`：必须存在
- `会话权限`：根据命令参数自动推断展示（如 `--sandbox`、`--add-dir`）

### 错误与边界情况

- 未创建会话时发送：会提示先新建会话。
- 消息为空：会被拦截。
- 工作目录不存在：返回明确错误。

### 与 CLI 的关系

- 基本等价于执行：`codex exec <PROMPT>`
- GUI 自动管理会话列表、历史消息和右侧运行可视化。

## 场景 2：任务运行中继续提问（排队）

### 功能目标

避免“上一条未完成时无法继续输入”的阻塞。

### 操作步骤

1. 发送一条较长请求。
2. 在任务运行中再发送新消息。
3. 观察状态栏队列数量变化，等待自动串行执行。

### 参数/配置说明

- 队列是“按会话隔离”的。
- 队列中的消息会按发送顺序执行。

### 错误与边界情况

- 若队列消息启动失败，会记录到结构化事件中。

### 与 CLI 的关系

- CLI 原生不提供多条消息队列管理；GUI 提供增强调度能力。

---

## 场景 3：失败后重试上一条

### 功能目标

在失败后快速复用上一条用户消息进行重试。

### 操作步骤

1. 任务失败后点击“重试上一条”。
2. 系统会基于上一条用户消息再次执行。

### 参数/配置说明

- 重试会进入“新会话重试”语义（避免复用失效会话 ID）。

### 错误与边界情况

- 若当前仍在运行：不可重试。
- 若历史中没有用户消息：会提示不可重试。

### 与 CLI 的关系

- CLI 需手工复制上一条输入重跑；GUI 提供一键重试。

---

## 场景 4：查看运行细节（结构化事件 / 运行步骤 / JSON）

### 功能目标

提供可审计、可回溯的执行过程，而不只看最终回复。

### 操作步骤

1. 打开右侧面板。
2. 在三个标签之间切换：
   - 结构化事件
   - 运行步骤
   - 事件原文(JSON)
3. 在“运行步骤”中按需展开具体步骤。

### 参数/配置说明

- 运行步骤当前默认折叠，可逐条展开。

### 错误与边界情况

- “清空运行日志”在会话运行中默认会阻止（避免误清）。

### 与 CLI 的关系

- CLI 有原始输出；GUI 提供三种视角并行展示（增强）。

---

## 场景 5：会话管理（重命名 / 关闭 / 清空内容）

### 功能目标

对多会话进行长期管理，降低上下文切换成本。

### 操作步骤

1. 左侧会话列表切换会话。
2. 使用“重命名”修改会话名。
3. 使用菜单 `对话` 进行关闭/清空等操作。

### 参数/配置说明

- 关闭最后一个正在运行的会话会被保护，避免丢失运行上下文。

### 错误与边界情况

- 清空对话内容时若任务运行中，会提示先停止。

### 与 CLI 的关系

- CLI 通常以单会话命令驱动；GUI 提供会话生命周期管理（增强）。

---

## 场景 6：视图与显示控制

### 功能目标

在“信息密度”和“阅读舒适度”之间快速切换。

### 操作步骤

1. 调整“对话字号”（滑杆或数字输入）。
2. 菜单 `视图` 中切换：
   - 隐藏/显示配置信息
   - 隐藏/显示右侧面板
   - 隐藏/显示左侧会话
3. 使用菜单 `视图 -> 放大/缩小` 调整整体窗口缩放。

### 参数/配置说明

- `对话字号` 与 `视图缩放` 为独立控制，最终视觉效果是叠加。

### 错误与边界情况

- 无。

### 与 CLI 的关系

- CLI 无 GUI 视图层，相关能力仅 GUI。

---

## 场景 7：运行中关闭窗口

### 功能目标

避免误关闭导致任务中断。

### 操作步骤

关闭窗口时，若存在运行任务，会弹出确认：

1. 取消
2. 停止任务并关闭
3. 直接关闭

### 参数/配置说明

- “停止任务并关闭”会先发 stop 请求并等待短时间。

### 错误与边界情况

- 强制关闭可能造成当轮输出不完整。

### 与 CLI 的关系

- CLI 中断通常由终端/进程控制；GUI 提供关闭保护（仅 GUI）。
